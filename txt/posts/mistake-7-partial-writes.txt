  SPECIAL -- IPC MISTAKE: PARTIAL WRITES
  Mistake 7: Partial writes
  ============================================================================================


  write() looks like it either writes everything or fails. It does neither. It returns the
  number of bytes it actually managed to write, and that number can be smaller than what was
  asked for -- with no error, no warning, just a quiet partial success the caller has to check
  for.

  --------
  FIRST, WHAT A PARTIAL WRITE LOOKS LIKE
```text
[process] --write(100 bytes)--> [kernel send buffer, only 40 bytes free]
                                        |
                               returns 40 (a partial write)
                                        |
                  the remaining 60 bytes were never written -- caller must loop
```
  Correct code loops until bytes_written reaches total_len. Code that trusts write's return
  value to always equal what it asked for will silently drop the tail of every write that
  doesn't fit in one pass.

  --------
  THEN THE ACTUAL CODE THIS RUNS ON

```text
#include <stdio.h>
#include <iostream>
#include <cstring>
#include <fstream>
#include <fcntl.h>
#include <unistd.h>

void support() {
    // BLANK: Write "hello consumer" to a file named "chat.txt"

    int fd = open("chat.txt", O_WRONLY | O_CREAT, 0644); 
    if (fd == -1) {
        perror("open failed");
        return;
    }

    char buffer[256]; 
    int len = strlen("hello consumer");
    strcpy(buffer, "hello consumer"); 
    write(fd, buffer, len);


    close(fd);
    
}

void consumer(){
    std::cout << "hello; this is consumer";
}

int main(int argc, char* argv[]) {
    if (argc < 2) {
        printf("Usage: ./v1 <support|consumer>\n");
        return 1;
    }
    
    // BLANK 1: Check argv[1] for "support"
    // MISTAKE REMINDER: strcmp returns 0 on match! 'if (strcmp(..))' is false on match.
    if ( strcmp(argv[1], "support") == 0 ) {
            support();
            }

    
    // BLANK 2: Check argv[1] for "consumer"
    // MISTAKE REMINDER: Need '== 0' to run on match!
    if ( strcmp(argv[1] , "consumer") == 0 ) { 
        consumer();
        }
    return 0;
}
```

  --------
  SO WHY write() CAN RETURN LESS THAN ASKED
  write() is a syscall handing bytes to a kernel-owned buffer, not a guarantee the bytes reach
  their destination. If that buffer has less free space than the request, the kernel copies
  what fits, returns that smaller count, and leaves the rest for the caller to notice and
  retry. Nothing about the call fails -- the return value is the only signal that anything was
  incomplete.

  A loop that checks bytes_written == total_len every time closes that gap. Code that assumes
  one write() call finishes the job will, sooner or later, silently ship a truncated payload.

  >> STANDARD NAMES
     write() returning less than requested -> a partial write
     the loop that makes writes reliable    -> loop until bytes_written == total_len
