  SPECIAL -- IPC MISTAKE: THE GHOST CONNECTION ECONNABORTED ACCEPT BUG
  Mistake 12: The Ghost Connection ECONNABORTED accept bug
  ============================================================================================


  A client can connect and disconnect entirely before the server ever calls accept(). The
  handshake the client experienced was completely real, and the server still gets nothing
  back but a dead end.

  --------
  FIRST, WHERE THE CONNECTION ACTUALLY DIES
```text
client connects ---> TCP handshake (OS kernel queue) ---> client Ctrl+C / RST
                                                                |
server accept() <--------------------------------------- ECONNABORTED / returns -1
       |
  write(-1) ---> server dies or fails silently
```
  The OS completes the TCP handshake before the server's code ever calls accept() -- the
  kernel queue holds the finished connection until the server is ready to collect it. If the
  client connects and immediately kills itself (Ctrl+C, or a raw RST) before that collection
  happens, the connection dies sitting in the queue. accept() still returns, but with -1, not
  a valid descriptor. Skip the -1 check and the server calls write(-1, ...) next, which fails
  silently or crashes depending on the platform. On a blocking accept(), the fix is checking
  the return value; O_NONBLOCK is the separate fix for not freezing the whole thread waiting
  on the next client.

  --------
  THEN THE ACTUAL CODE THIS RUNS ON -- COMPILE IT AND RUN IT YOURSELF

  No Makefile, no CMake, one command:
```text
g++ -std=c++17 -o demo mistake-12-ghost-connection-econnaborted.cpp
./demo support     -- terminal 1, sleeps 10s, then calls accept()
./demo consumer    -- terminal 2, connects then immediately RSTs
```

```cpp
#include <stdio.h>
#include <iostream>
#include <cstring>
#include <unistd.h>
#include <sys/socket.h>
#include <netinet/in.h>
#include <arpa/inet.h>

void support() {
    int server_fd = socket(AF_INET, SOCK_STREAM, 0);
    int opt = 1;
    setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, (const char*)&opt, sizeof(opt));

    struct sockaddr_in address;
    address.sin_family = AF_INET;
    address.sin_addr.s_addr = INADDR_ANY;
    address.sin_port = htons(8080);

    bind(server_fd, (struct sockaddr*)&address, sizeof(address));
    listen(server_fd, 10);

    std::cout << "Support (BLOCKING) started." << std::endl;
    std::cout << "Sleeping for 10s... While I sleep, the consumer will connect and send a brutal RST." << std::endl;

    sleep(10);

    std::cout << "\nAwake! Calling blocking accept()..." << std::endl;
    int client_fd = accept(server_fd, nullptr, nullptr);

    std::cout << "accept() returned FD: " << client_fd << std::endl;

    if (client_fd != -1) {
        int bytes = write(client_fd, "hello consumer", 14);
        std::cout << "write() returned: " << bytes << std::endl;
        close(client_fd);
    } else {
        perror("accept failed");
    }

    close(server_fd);
    std::cout << "Support exiting." << std::endl;
}

void consumer() {
    int client_fd = socket(AF_INET, SOCK_STREAM, 0);
    struct sockaddr_in address;
    address.sin_family = AF_INET;
    address.sin_port = htons(8080);
    inet_pton(AF_INET, "127.0.0.1", &address.sin_addr);

    std::cout << "Consumer connecting..." << std::endl;
    if (connect(client_fd, (struct sockaddr *)&address, sizeof(address)) == -1) {
        perror("connect failed");
        return;
    }
    std::cout << "Consumer connected! In queue." << std::endl;

    // TRICK: Force an RST instead of a clean FIN when we close
    struct linger sl;
    sl.l_onoff = 1;
    sl.l_linger = 0;
    setsockopt(client_fd, SOL_SOCKET, SO_LINGER, (const char*)&sl, sizeof(sl));

    std::cout << "Consumer sending brutal RST (Ctrl+C simulation) and dying..." << std::endl;
    close(client_fd);
}

int main(int argc, char* argv[]) {
    if (argc < 2) { printf("Usage: ./demo <support|consumer>\n"); return 0; }
    if (strcmp(argv[1], "support") == 0) support();
    if (strcmp(argv[1], "consumer") == 0) consumer();
    return 0;
}
```

  Running this prints:

      Usage: ./demo <support|consumer>

  --------
  SO WHAT THE REAL COMMIT SAYS ABOUT THIS
  commit 9e354c2654cdff1edf15f0ef144a59cf5c94f48d, "The CLOSE_WAIT Zombie Apocalypse (EMFILE
  bug)" -- a separate but related discovery from the same debugging session: omitting
  close(client_fd) on the server leaves the socket in CLOSE_WAIT, permanently holding a file
  descriptor even after a clean consumer disconnect. A stress test firing 5000 clean consumers
  at the server accepted exactly 2047 connections (fd 4 through 2050) before crashing on the
  2048th -- the default Microsoft C runtime's hard cap on low-level I/O descriptors per
  process. One missing close() is a slow-burning fuse toward that ceiling.

  --------
  SO A GHOST CONNECTION IS A RACE, NOT A LOGIC ERROR
  Nothing in the server's own code is wrong when this happens. The client and the kernel
  finished a real handshake, and the client tore it down again, all before the server's thread
  got scheduled to run accept(). The fix is not preventing the race -- it cannot be prevented
  -- it is treating -1 from accept() as a routine, expected outcome instead of an impossible
  one.

  >> STANDARD NAMES
     a connection dead before accept() collects it -> a ghost connection
     the error accept() returns for it              -> ECONNABORTED / -1
