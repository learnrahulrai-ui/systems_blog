  GLOSSARY -- plain term, then the textbook name

  This blog builds each thing from what it does before sticking a label on it. The labels are
  gathered here, so a term you meet in a paper or a man page maps back to the plain words used
  in the posts.

  send-window / how much the kernel lets you push before it makes you wait
      -> TCP congestion window (cwnd)

  the point where the kernel stops doubling the window and starts adding slowly
      -> slow-start threshold (ssthresh)

  the kernel remembering a route's speed and reusing it for the next connection on that route
      -> TCP Control Block Interdependence (RFC 2140)

  kernel-side memory that can never be swapped to disk
      -> non-paged pool

  a socket that refuses to make your thread wait, returning "not ready" instead
      -> non-blocking socket (O_NONBLOCK / EAGAIN)

  the "too many open files" refusal
      -> EMFILE (per-process file-descriptor limit reached)

  waiting on many sockets at once with one call
      -> readiness multiplexing (poll / epoll / IOCP)

  the receiver being full pushing back on the sender until it slows down
      -> backpressure

  handing the network card a buffer without copying it again first
      -> zero-copy

  the CPU dropping one job and picking up another, saving and reloading its state
      -> context switch

  the CPU asking for memory that is not resident yet, so the kernel fetches it
      -> page fault

  a closed connection kept around a while so late packets do not confuse a new one
      -> TIME_WAIT

  holding back tiny sends to bundle them into one bigger packet
      -> Nagle's algorithm

  the largest chunk a link will carry without splitting it
      -> MTU (maximum transmission unit)

  the small integer the kernel hands you to name an open file or socket
      -> file descriptor (fd)

  a single trip from your program into the kernel and back
      -> system call (syscall)
